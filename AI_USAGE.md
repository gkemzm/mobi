## 사용한 AI도구와 사용 이유

### chatGpt

AI활용은 다음과 같은 점을 생각하여 주로 GPT를 통하여 했습니다.

1. 과제의 양이 실제 프로젝트의 양만큼 방대하지 않습니다.
2. 난이도 높은작업이 없어 클로드 코드의 심층적인 기능이 필수가 아닙니다.
3. Custom GPT를 활용해 schma, skill, 행동방식, 제한사항을 프롬프트로 작성해 사전학습을 시킬 수 있습니다.
4. 파일(db.json)을 사전에 저장하여 활용하기에 3번 내용과 시너지를 낼 수 있습니다.

이러한 이유로 이번 과제에 있어 높은 효율을 보여준다고 판단했기 때문에 GPT를 Custom한 모델을 사용 했습니다.

### cursor

코드 일관성, 효율, 등 에디터 내부 코드에 대한 분석을 위하여 사용했습니다.

## 프롬프트, 의사결정 과정, 컨텍스트 파일에 대한 기술 등

### 사용 프롬프트

... 제공된 schma 내용

- data File: db.json

- language: JS + TS
- library: React 18
- framework: next.js14.2.2 (Main)
- css: scss

작동방식: CO-STAR Structure (MAIN)

Context (배경)
AI가 문제를 이해할 수 있도록 상황 설명을 간결하게 작성.

Objective (목표)
구체적으로 어떤 결과를 얻고 싶은지 명시.

Style, Tone = 자유

Audience
작성자 본인 (면접 과제 진행중)

Response format

1. 위 내용을 참조
2. 물어본 것만 대답하기(부가적인것 X)
3. 질문시 Context와 Objective를 알려줘서 올바른 질문을 했는지 검토하게해주기
4. scss 요청시 반응형(1024px) 고려하기
5. Next.js 14.2.2 를 기준으로 답변하기
6. api및 복잡한 알고리즘은 검증 하기
7. 1안을 작성후 다시 한번 깊게 생각하기

예시:

대제목(굵게 표시)

소제목 - 설명

소제목 - 예시 코드 블록

요청시 아래의 구조로 답변

ex) 체인형 구조로 다시 설명해줘 => 아래구조로 실행 (그전엔 미실행)

Chain of Thought Structure (SUB)

문제 정의: "내가 원하는 정보는 무엇인가?"

접근 방식: "Git 명령어를 주제별로 분류"

단계별 해결: "각 명령어 → 옵션 설명 → 예시 코드"

검증: "빠진 주요 명령어는 없는지 확인"

대안 탐색: "비슷한 명령어가 있다면 추가로 제공"

### 의사결정 과정

저는 화면예시를 기반으로 실제 서비스를 가정하고 작업을 진행하였습니다.

그 과정에서 1차적으로 생각한 기획에 맞춰 화면을 설계했고 이후 AI를 활용해 보강하는 방식으로 진행했습니다.

이러한 방향성을 잡아 AI가 좀 더 명확한 Context와 Objective 를 도출 할 수 있게 틀을 잡고

AI안을 참고하며 과제를 진행하였습니다.
